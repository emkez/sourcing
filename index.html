# Инструкция для Claude: извлечение компаний и контактов рекрутеров из `messages.html`

Твоя задача — обработать HTML-экспорт сообщений и собрать Excel-файл для внутренней команды.

## Цель
Нужно достать **списком компании** из сообщений, где есть:
- название компании;
- контакт рекрутера;
- при необходимости — ссылка на вакансию;

Также нужно на отдельный лист вынести сообщения, где:
- название компании **спрятано намеренно** или замаскировано;
- при этом есть контакт рекрутера.

## Что считать подходящей записью
Запись включается в основной лист, если в сообщении есть минимум:
1. **Компания** — явное название компании, бренд, домен, ссылка на сайт, название работодателя или клиента.
2. **Контакт рекрутера** — Telegram, email, LinkedIn, форма отклика, ник, личный контакт, указанный как способ связи.

Если компания не названа напрямую, но есть признаки вроде:
- NDA;
- "product company", "US client", "large client", "stealth startup";
- ссылка без явного бренда;
- бренд зашит в домене, сокращении, картинке или тексте без явного поля Company;

то такую запись **не теряй** — выноси на отдельный лист `Вакансии для пробива`.

## Формат итогового Excel
Сделай workbook минимум с 3 листами:

### 1) Лист `Компании`
Основной реестр найденных компаний.

Столбцы:
1. Дата
2. Отправитель
3. Компания
4. Контакт рекрутера
5. Ссылка на вакансию
6. Позиция
7. Локация
8. Сайт компании (если нет в сообщении, то найди сам)
9. Линкедин компании (если нет в сообщении, то найди сам)
10. Размер компании (если нет в сообщении, то найди сам)

### 2) Лист `Спрятанные компании`
Сюда попадают сообщения, где компания не названа явно, но есть рекрутерский контакт и есть шанс вычислить компанию.

Столбцы:
1. Дата
2. Отправитель
3. Предполагаемая компания
4. Контакт рекрутера
5. Заметка

## Правила извлечения

### Название компании
Ищи компанию в таком приоритете:
1. Явное поле Company, Компания, We at <brand>.
2. Название в тексте вакансии.
3. Бренд в ссылке на сайт или домене.
4. Если явно не найдено — вынеси на лист Спрятанные компании.

### Контакт рекрутера
Нужно извлекать любые контакты:
- t.me/...
- @username
- email
- LinkedIn URL
- Google Form / Typeform / Ashby / Greenhouse / сайт отклика
Если контактов несколько — складывай через ; в одну ячейку.

### Удаление дублей
Дубль = Компания + Контакт рекрутера совпадают, ИЛИ тот же ID сообщения.
НО: если у одной компании разные контакты — это НЕ дубль.

## Технические требования к файлу
- один .xlsx файл
- автофильтры на всех листах
- закрепить верхнюю строку
- перенос текста для Фрагмент сообщения и Заметка
- подобрать ширину колонок
- отсортировать основной лист по дате, новые сверху

## Важные ограничения
- Не выдумывай компании, если уверенность низкая.
- Если компания не доказана — положи на лист Спрятанные компании.
- Не теряй сообщения только из-за того, что бренд замаскирован.
- Оставляй Фрагмент сообщения, чтобы человек мог быстро перепроверить вывод.

Во вкладке Perplexity заранее вставлен промпт анализа компании — при переходе к каждой новой строке просто вводишь название компании и получаешь готовую выжимку с ЛПР и контактами:

ПРОМПТ: анализ компании для аутрича (Perplexity) 
Мне нужно получать краткую выжимку по компаниям для аутрича.
На каждый мой ввод — это только название компании.
Твоя задача — найти и выдать строго в таком формате и только данные,
без пояснений и лишнего текста:

1. Тип компании
   сервисная / продуктовая / смешанная (одно слово или короткая фраза)

2. Страна регистрации (HQ)
   страна, город (если есть)

3. Сайт компании
   полный URL

4. LinkedIn компании
   название + URL или «нет данных», если не найдено

5. Количество сотрудников
   вилка/оценка (например, 11–50, 51–200, 201–500 и т.п.)

6. ЛПР (C‑level, тех, коммерция, русскоязычные, если есть)
   Список буллетами, по каждому:
   • Имя Фамилия — должность, страна/город (если есть) — LinkedIn URL

7. ЛПР по HR (HRD / Head of HR / HRBP / HR / Recruiter, русскоязычные, если есть)
   Список буллетами, по каждому:
   • Имя Фамилия — должность, страна/город (если есть) — LinkedIn URL

Если каких‑то данных нет, пиши «нет данных» на соответствующей строке,
а не пропускай строку.
Никаких вводных фраз, пояснений, комментариев, выводов —
только сухие данные по структуре выше.
💡

Промпт вводится один раз при открытии вкладки. Дальше просто печатаешь название компании — Perplexity сам подтягивает актуальные данные из сети.

📋

Данные из Perplexity (ЛПР, сайт, LinkedIn, размер компании) заносишь в карточку компании в AmoCRM — в соответствующие поля сделки.

05
CRM-проверка
~1 мин

Проверка и создание сделки в AmoCRM

Название компании вбиваешь в поиск по CRM и смотришь историю:

Если история ≥ 300 дней
Создаёшь новую сделку, заполняешь с нуля. Старую ссылку скидываешь Диме на удаление.
Если история свежая
Работаешь в существующей сделке, обновляешь данные.
Что делать со скрытыми компаниями (лист «Спрятанные компании»)

06
Telegram-поиск
~1 мин

Поиск по всем чатам в Telegram

Открываешь Telegram и через глобальный поиск по сообщениям ищешь название компании. Видишь сразу все вакансии из 50+ чатов, отсортированные по дате — можно быстро оценить активность и свежесть постингов.

💡

Это позволяет быстро понять, насколько активно компания публикует вакансии и нет ли уже более актуального постинга, который не попал в выгрузку.

07
Первый контакт
~2 мин

Прописываешь ЛПР, который постил вакансию

Находишь аккаунт рекрутера / ЛПР в Telegram и отправляешь первое сообщение. Персонализируй имя и название вакансии под конкретный случай:

Шаблон первого сообщения — Telegram
[Имя], добрый день! Я представляю hr-агентство GBagency (https://www.gbhiring.com/), увидел в чате что вы в поиске [должность из вакансии] У нашей команды богатый опыт закрытия разнообразных сложных вакансий, выработана большая база сильных it-специалистов по всему миру (более 50.000 человек). Хотел предложить возможность сотрудничества) Было бы интересно обсудить это?
✏️

Обязательно замени [Имя] и [должность из вакансии] — персонализация критична для конверсии.

08
LinkedIn + Habr
~5 мин

Расширенный поиск новых ЛПР

Параллельно ищешь дополнительные контакты в двух источниках:

LinkedIn
Находишь ЛПР → отправляешь запрос на коннект → пишешь после принятия в друзья.
Habr Career
Ищешь телефоны и Telegram-контакты → пишешь сразу напрямую.
⚡

В LinkedIn — только после принятия коннекта. В Habr — можно сразу, так как контакты публичные.

09
Итерация и цикл

Следующая позиция → следующий чат

Обработав одну компанию — переходишь к следующей строке в таблице от Claude, повторяешь шаги 5–8. Когда таблица по текущему чату отработана полностью — возвращаешься к шагу 2, экспортируешь следующий чат из списка и начинаешь заново.

компания готова → следующая строка → таблица закончилась? → следующий чат → экспорт → Claude
